Study Room Prototype (Printer Friendly)

Clean black-and-white layout for printing and review notes.

1) Auth Landing

Study Room Portal title
Tabs: Login | Register
Email + Password input

Actions: Login, Go Register

2) Auth Errors

Error: Wrong password
Error: User not found
Recovery: Reset password or Register

Actions: Try Again, Forgot Password

3) Forgot Password

Enter account email
Validation: required + valid email
Result: reset link sent / email not found / link expired

Actions: Send Reset Link, Back to Login

4) Register View

Name, Email, Password, Role
Success message and redirect

Actions: Register, Back to Login

5) Registration Errors

Missing required fields
Invalid email format
Password too short (< 5 symbols)

Actions: Fix Inputs, Submit Again

6) Dashboard

Welcome + Role badge
Study room list/cards
Create/Join room actions

Actions: Open Room, Logout

7) Time Choosing

Pick date, start, end
Duration preview

Actions: Check Availability, Confirm Slot

8) Booking Conflict Error

Conflict: room date/time already booked
Conflicting slot details + suggested alternatives

Actions: Pick Another Time, Choose Different Room

9) Reserved Rooms

Bookings list + room timeslot
Status: upcoming / active / completed
Quick actions: edit / cancel

Actions: View Details, Reserve Another

10) Validation & System Errors

Missing fields / invalid time range
Session expired
Network/server error with retry hint

Actions: Retry, Back to Dashboard

Printer-Friendly Prototype Testing Scenarios

  • Login with valid student account
  • Login with wrong password
  • Login with unknown email (user not found)
  • Forgot password with valid email
  • Forgot password with unknown email
  • Register with valid inputs
  • Register with missing fields
  • Register with invalid email format
  • Register with password shorter than 5 symbols
  • Choose an available room date/time and confirm
  • Try booking an already reserved room timeslot
  • Open Reserved Rooms and verify booking status
  • Edit a reservation and re-check availability
  • Cancel a reservation and verify list refresh
  • Simulate expired session and verify redirect/login prompt
  • Simulate network/server error and verify retry UX
